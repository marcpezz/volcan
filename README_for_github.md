# Volcanic Thermal Simulation Framework

This framework enables running ~1000 thermal simulations across **10 volcanic intrusion scenarios** using a Python-Julia hybrid workflow.

## Overview

**Python** handles:
- Scenario generation with Latin Hypercube Sampling
- Parameter sampling and JSON export
- Summary CSV generation

**Julia** handles:
- Thermal simulations using MagmaThermoKinematics.jl
- Physics-based PDE solving
- HDF5 snapshot export for post-processing and animation
- Eruptibility and thermal diagnostics

## Directory Structure

```text
.
├── python/
│   └── 01_generate_scenarios.py   # Generate 1000 scenario JSON files
├── julia/
│   ├── run_animation_ML.jl        # Main simulation and HDF5 snapshot export
│   ├── animate_from_h5.jl         # Rebuild GIF animation from HDF5 snapshots
│   └── eruptibility_metrics.jl    # Eruptibility calculations
├── scenarios/                     # Generated scenario JSON files
└── results/                       # HDF5 output files
```

## Workflow

### Step 1: Generate Scenarios

Generate **1000 scenarios total** using Latin Hypercube Sampling:
- **100 samples per scenario family**
- **10 scenario families**

```bash
cd python
python 01_generate_scenarios.py
```

This creates:
- `scenarios/scenario_XXXX_SY.json` - Individual scenario files
- `scenarios/scenarios_summary.csv` - Summary of all scenarios

### Step 2: Run Simulations

Run a single scenario on your local machine:

```bash
julia --project=. julia/run_animation_ML.jl scenarios/scenario_0300_S4.json
```

This produces an HDF5 file such as:

```text
results/ML_data_300_S4.h5
```

### Step 3: Recreate an Animation from HDF5

```bash
julia --project=. julia/animate_from_h5.jl results/ML_data_300_S4.h5 10
```

## Scenario Families Generated by `01_generate_scenarios.py`

### S1: Deep Dike Intrusions (Lower Crust)
- **Depth center**: 20-30 km
- **Flux rate**: 0.001-0.05 km3/yr
- **Pattern**: steady
- **Depth window**: lower crust
- **References**: Annen & Sparks 2002; Annen et al. 2006

### S2: Mid-Crustal Dike Intrusions
- **Depth center**: 10-20 km
- **Flux rate**: 0.001-0.03 km3/yr
- **Pattern**: steady
- **Depth window**: upper crust
- **References**: Annen et al. 2009; Glazner et al. 2021

### S3: Shallow Dike Intrusions (Upper Crust)
- **Depth center**: 5-15 km
- **Flux rate**: 0.001-0.03 km3/yr
- **Pattern**: steady
- **Depth window**: upper crust
- **References**: Annen et al. 2006; Cashman et al. 2017

### S4: Transcrustal Dike System
- **Deep reservoir depth**: 20-30 km
- **Shallow reservoir depth**: 5-10 km
- **Deep flux**: 0.005-0.05 km3/yr
- **Shallow flux**: 0.001-0.02 km3/yr
- **Flux ratio**: 0.2-0.8
- **Pattern**: transcrustal
- **Depth window**: transcrustal
- **References**: Cashman et al. 2017; Liu & Lee 2020

### S5: Two-Stage System (Deep -> Shallow)
- **Stage 1 depth**: 20-30 km
- **Stage 2 depth**: 5-15 km
- **Stage 1 flux**: 0.005-0.05 km3/yr
- **Stage 2 flux**: 0.001-0.02 km3/yr
- **Transition time**: 0.2-0.8 Myr
- **Pattern**: two_stage
- **Depth window**: two_stage
- **References**: Caricchi et al. 2014; Liu & Lee 2020

### S6: GENEVA Lower Crust (Radial Accretion)
- **Depth center**: fixed at 25 km
- **Sill diameter**: 10-30 km
- **Sill thickness**: 50-100 m
- **Flux rate**: 7e-6 to 15e-6 km3/yr/km2
- **Pattern**: radial_accretion
- **Depth window**: lower crust
- **References**: Annen et al. 2015; Karakas et al. 2017

### S7: GENEVA Upper Crust (Radial Accretion)
- **Depth center**: fixed at 8 km
- **Sill diameter**: 5-15 km
- **Sill thickness**: 30-80 m
- **Flux rate**: 10e-6 to 20e-6 km3/yr/km2
- **Pattern**: radial_accretion
- **Depth window**: upper crust
- **References**: Annen et al. 2015; Karakas et al. 2017

### S8: UCLA Lower Crust (Elliptical Sills)
- **Depth center**: fixed at 25 km
- **Sill semi-major axis**: 2-5 km
- **Sill aspect ratio**: 2-5
- **Sill volume**: 5-20 km3
- **Pattern**: elliptical_sills
- **Depth window**: lower crust
- **References**: Dufek & Bergantz 2005; Annen et al. 2015

### S9: UCLA Upper Crust (Elliptical Sills)
- **Depth center**: fixed at 8 km
- **Sill semi-major axis**: 1-3 km
- **Sill aspect ratio**: 1.5-3
- **Sill volume**: 2-10 km3
- **Pattern**: elliptical_sills
- **Depth window**: upper crust
- **References**: Dufek & Bergantz 2005; Annen et al. 2015

### S10: Episodic Pulses (Mid-Crust)
- **Depth center**: 10-20 km
- **Active pulse duration**: 50-100 kyr
- **Quiet duration**: 100-300 kyr
- **Number of pulses**: 3-5
- **Pattern**: episodic_pulses
- **Depth window**: mid crust
- **References**: Caricchi et al. 2014; Barboni et al. 2016

## Common Parameter Ranges

These ranges are sampled for all scenario families unless overridden by scenario-specific settings.

- **Magma temperature**: 900-1200 C
- **Injection interval**: 500-50,000 years
- **Dike thickness**: 10-500 m
- **Intrusion width**: 1-20 km
- **Host conductivity**: 2.0-4.0 W/m/K
- **Host heat capacity**: 800-1200 J/kg/K
- **Geotherm gradient**: 20-25 K/km
- **Simulation duration**: fixed at 1.0 Myr

## Derived Simulation Setup

The scenario generator also computes the following derived settings:

- **Domain width**: fixed at 40 km
- **Domain depth**: fixed at 30 km
- **Grid resolution**: 200 m nominal spacing
- **Grid size**: 200 x 150 cells
- **Time conversion helpers**:
  - `magma_temp_K`
  - `injection_interval_s`
  - `intrusion_width`
  - `simulation_duration_s`

## Output Contents

Each HDF5 result file can contain:

- Grid coordinates
- Parameter metadata
- Thermal snapshots through time
- Melt fraction snapshots through time
- Phase fields
- Heat flux fields
- Conductivity, heat capacity, and latent heat fields
- Tracer exports for petrological post-processing
- Snapshot-level thermal and eruptibility statistics

## Performance Notes

- **Grid resolution**: fixed at 200 m in the current generator
- **Grid size**: 30,000 cells per model
- **Total dataset target**: 1000 simulations
- **Runtime and file size** depend strongly on scenario type, snapshot interval, and hardware

## Customization

### Modify Parameter Ranges

Edit `python/01_generate_scenarios.py`:

```python
common_params = {
    'magma_temp_C': (900, 1200),
    ...
}
```

### Change the Number of Samples

Edit the generator initialization:

```python
generator = ScenarioGenerator(n_samples_per_scenario=100, seed=42)
```

### Modify Snapshot Density

When running Julia, pass a smaller frame interval to save more snapshots:

```bash
julia --project=. julia/run_animation_ML.jl scenarios/scenario_0000_S1.json 10
```

### Add a New Scenario Family

Add it in `scenario_params` and `scenario_definitions` in `python/01_generate_scenarios.py`.
